You'll see this tool called Clawdbot, OpenClaw, and Moltbot on YouTube - they're all the same thing! The name changed due to trademark issues, but the functionality is identical.

ü§î What is Clawdbot?

Clawdbot is your personal AI gateway daemon - think of it as a bridge that connects powerful AI models (Claude, GPT, Gemini, local models) to your everyday messaging apps. Instead of going to a website to chat with AI, you can message it directly on Telegram, Discord, Slack, WhatsApp, or even Signal!

It runs on your computer (or a Raspberry Pi) 24/7, giving you an always-available AI assistant that can read files, browse the web, control smart home devices, send emails, and much more through MCP (Model Context Protocol) integrations.

ü§î What Are Skills & MCP?

Skills and MCP (Model Context Protocol) are what turn a basic AI chatbot into a powerful automation agent. They let AI actually DO things instead of just talking about them.

Think of it this way: Without Skills/MCP, AI is like a smart person locked in a room with no phone or computer. WITH Skills/MCP, that same person suddenly has access to your files, email, databases, APIs, browser, and more!

üîå MCP - Model Context Protocol

Created by Anthropic ‚Ä¢ The "USB-C for AI"

What is MCP?

MCP is a universal standard for connecting AI models to external tools, data sources, and APIs. Instead of every AI having different integration methods, MCP provides ONE protocol that works everywhere.

How Does It Work?

üñ•Ô∏è MCP Host - The AI application (Claude Desktop, Cursor, etc.)

‚ö° Claude Code Skills

Reusable prompts & workflows for Claude Code CLI

What Are Claude Code Skills?

Skills are pre-written prompt templates that Claude Code can use for specific tasks. Instead of writing the same complex prompts over and over, you save them as Skills and reuse them!

How Skills Work

Skills are stored as SKILL.md files in your project or global config:

~/.claude/skills/my-skill/SKILL.md

Example Skill Structure

# SKILL.md - Code Reviewer

## Description
Review code for bugs, security issues, and best practices.

## Instructions
1. Analyze the provided code
2. Check for security vulnerabilities
3. Suggest performance improvements
4. Ensure coding standards are followed

## Output Format
- üêõ Bugs found
- üîí Security issues  
- ‚ö° Performance tips
- ‚ú® Best practice suggestions
